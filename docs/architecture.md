# 架构说明

## 四层流水线

系统采用四层解耦架构：

```
┌─────────────────────────────────┐
│   第一层：特征提取层            │
│   (features/note_features.py)   │
│   音符级特征向量                │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│   第二层：维度计算层            │
│   (analysis/ + dimensions/)     │
│   滑窗分析 → 七维度独立计算     │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│   第三层：聚合层                │
│   (aggregation/)                │
│   加权聚合 → 三类目标难度       │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│   第四层：标定层                │
│   (normalization/)              │
│   规则标准化 → 可外部标定       │
└─────────────────────────────────┘
```

## 模块依赖关系

```
models (数据模型) ← 被所有模块依赖
  ↑
parsers → 输入 → models
  ↓
features ← models
  ↓
analysis ← features
  ↓
dimensions ← features + analysis
  ↓
aggregation ← dimensions 的输出
  ↓
explanation ← 所有层输出
  ↓
engine ← 编排所有层
  ↓
cli ← engine
```

## 关键设计决策

### 维度独立性
每个维度是 `BaseDimension` 的独立子类，可单独实例化和测试。新增维度只需：
1. 创建新的子类文件
2. 在 `__init__.py` 注册
3. 在聚合配置中添加权重

### 聚合可配置性
`AggregationProfile` 以配置对象形式定义权重，不硬编码在计算逻辑中。可通过参数注入替换。

### 标准化分离
原始特征值和标准化分数分开存储在 `DimensionScore` 中。标准化参数集中配置在 `NormConfig` 中。

### 数据驱动预留
- `NormConfig` 支持外部加载
- 聚合权重可参数化
- 维度计算基类支持替换
- 结果包含原始值和详情字典
