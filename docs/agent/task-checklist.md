# Task Checklist（Agent 执行清单）

## A. 代码改动前

- [ ] 确认需求范围（算法 / API / 前端 / 部署）
- [ ] 确认是否属于“必要功能（MVP）”；非必要项默认不做
- [ ] 读取对应模块文档（本目录）
- [ ] 确认是否影响返回字段或部署策略

## B. 代码改动后

- [ ] Python 逻辑变更：运行 `pytest tests/ -v`
- [ ] 前端变更：运行 `cd webui && npm run build`
- [ ] 部署变更：检查 `deploy/ubuntu/*` 配置一致性
- [ ] 文档变更：同步 `README.md` 与 `docs/`

## C. 提交说明建议

- [ ] 写清“改动范围 + 原因 + 兼容性影响”
- [ ] 明确是否变更 API 字段
- [ ] 明确是否需要重新构建前端或重启服务
- [ ] 标注本次是否仅完成 MVP 必要范围

## D. 禁止项

- [ ] 无需求时删除已有 API 字段
- [ ] 在前端复制后端评分逻辑
- [ ] 修改与任务无关的大段代码
